## 2. 배경 및 목적 (Background & Goals)

### 2.1 프로젝트 배경

당사는 Meta 광고를 통해 매일 잠재고객(리드) 데이터를 수집하고, 이를 컨설턴트에게 배분하여 영업 활동을 진행하고 있습니다. 현재 이 과정에서 여러 유료 도구를 복합적으로 사용하고 있어 비용과 복잡성 문제가 발생하고 있습니다.

#### 현재 시스템 구조 및 문제점

**현재 흐름:** Meta 광고 관리자 → 구글 시트 → Zapier (유료) → Flow.team (유료)

- Zapier + Flow.team 월 구독료 발생 (이중 비용)
- 구글 시트 → Flow.team 자동 동기화를 위해 유료 서비스 필수
- 컨설턴트가 구글 시트와 Flow.team에 결과를 이중 입력해야 함
- 사무직원이 여러 플랫폼을 오가며 수동으로 현황 집계
- 광고 캠페인별 성과 추적이 별도 작업으로 필요
- 실시간 현황 파악 불가로 의사결정 지연
- 팀 간 협업 (미팅 동행 등) 요청이 구두/메신저로 이루어져 추적 어려움
- **리드 등급 분류를 매번 수동으로 해야 함**

### 2.2 프로젝트 목적

Zapier와 Flow.team을 대체하는 자체 웹 플랫폼을 구축하여 유료 서비스 비용을 절감하고, 리드 관리의 전 과정을 단일 시스템으로 통합합니다.

#### 목표 시스템 구조

**개선 흐름:** Meta 광고 관리자 → LeadFlow (자체 플랫폼) → 완료

### 2.3 성공 지표 (KPI)

| 지표                    | 현재                      | 목표                   |
| ----------------------- | ------------------------- | ---------------------- |
| 월간 툴 비용            | Zapier + Flow.team 구독료 | 0원 (자체 플랫폼)      |
| 리드 배분 소요 시간     | 2-3시간/일                | 30분 이내/일           |
| 데이터 입력 횟수        | 2회 (구글시트+Flow.team)  | 1회 (LeadFlow만)       |
| 실시간 현황 파악        | 1일 지연                  | 실시간                 |
| 광고 ROI 분석           | 월 1회 수동 집계          | 실시간 대시보드        |
| 미팅 동행 요청 추적     | 불가능                    | 100% 추적 가능         |
| **등급 분류 소요 시간** | **수동 (건당 검토)**      | **자동 (업로드 즉시)** |

### 2.4 프로젝트 범위

#### 포함 사항 (In Scope)

- Meta 광고 리드 데이터 업로드 및 관리
- **등급 규칙 설정 및 자동 등급 분류**
- 관리자의 팀장(컨설턴트) 수동 배분 (드롭다운 선택)
- 팀 및 영업 관리자 관리
- 영업 관리자용 팀 대시보드
- 컨설턴트별 개인 페이지 (배분된 리드 확인 및 결과 입력)
- 미팅 동행 요청 기능
- 관리자용 실시간 현황 대시보드
- 광고 캠페인별 성과 추적
- 시스템 설정 기능 (등급 기준, 상태값 등 커스터마이징)

#### 제외 사항 (Out of Scope)

- Flow.team 연동 (대체가 목적이므로 제외)
- Zapier 연동 (대체가 목적이므로 제외)
- Meta 광고 캠페인 직접 운영/관리 기능
- 정산 및 급여 관리 시스템
- 모바일 앱 (1차 출시 기준)

### 3.3 사용자 페르소나

#### 페르소나 1: 사무직원/시스템 관리자 (박지영, 28세)

- **역할:** 매일 Meta에서 리드 다운로드, ~~등급 분류~~, 팀장에게 배분, 시스템 설정 관리
- **Pain Point:** 구글시트 → Zapier → Flow.team 연동 관리, 이중 작업, **매번 등급 수동 분류**
- **Needs:** 한 곳에서 업로드 → **자동 등급 분류** → 팀장 배분 → 현황 확인
- **기술 수준:** 상 (엑셀/구글 시트 고급 사용자)

#### 페르소나 2: 영업 관리자 (최영희, 40세)

- **역할:** 소속 팀의 팀장들 관리, 팀 전체 실적 모니터링, 팀원 코칭
- **Pain Point:** 팀원들 현황을 개별적으로 물어봐야 함, 팀 성과 취합 어려움
- **Needs:** 팀원별 리드 현황/실적 한눈에 확인, 팀 간 협업 현황 파악
- **기술 수준:** 중 (기본적인 PC 활용)

#### 페르소나 3: 팀장/컨설턴트 (김민수, 35세)

- **역할:** 배분받은 잠재고객에게 연락, 미팅/계약 성사
- **Pain Point:** 구글 시트 + Flow.team 이중 입력, 내 리드만 보기 어려움, 동행 요청 시 연락 번거로움
- **Needs:** 내 리드만 한눈에 확인, 결과 1회 입력으로 끝, 간편한 동행 요청
- **기술 수준:** 중 (새로운 툴 학습 의지 있음)

#### 페르소나 4: 경영진 (이철호, 45세)

- **역할:** 광고 성과 분석, 팀별/개인별 성과 관리, 전략 의사결정
- **Pain Point:** 데이터가 분산되어 통합 분석이 어려움
- **Needs:** 광고별 ROI, 팀별/개인별 성과, 전체 현황을 한눈에
- **기술 수준:** 중 (리포트 확인 위주)

### 3.4 사용자 여정 (User Journey)

#### AS-IS (현재 프로세스)

Meta 광고 관리자 → 리드 CSV 다운로드 → 구글 시트 업로드 → **수동 등급 분류 (건별 검토)** → 팀장별 시트에 복사 → Zapier가 Flow.team으로 동기화 → 컨설턴트가 구글 시트 + Flow.team 이중 입력 → 사무직원 수동 집계 → 동행 필요 시 카톡/전화로 개별 요청

#### TO-BE (개선 프로세스)

Meta 광고 관리자 → 리드 CSV 다운로드 → LeadFlow 관리자 페이지에 업로드 → **자동 등급 분류 (규칙 기반)** → 드롭다운으로 팀장 선택/배분 → 팀장 개인 페이지에서 확인 → 결과 1회 입력 → 미팅 동행 필요 시 시스템에서 요청 → 영업 관리자가 팀 현황 확인 → 관리자 대시보드에서 실시간 확인

### 4.2 기능별 상세 요구사항

#### F01. 리드 데이터 CSV 업로드 (자동 등급 분류 포함)

**설명:** 시스템 관리자가 Meta 광고 관리자에서 다운로드한 리드 CSV 파일을 업로드하면, 설정된 등급 규칙에 따라 자동으로 등급이 분류됨

**입력:** CSV 파일 (이름, 연락처, 업체명, 업종, 연매출, 종업원수, 신청일시, 광고 캠페인명 등)

**출력:**

- 업로드 완료 메시지
- **자동 등급 분류 결과 요약 (A등급 N건, B등급 N건, ...)**
- 전체 리드 목록에 데이터 추가

**프로세스:**

```
CSV 업로드 → 데이터 검증 → 중복 체크 → 등급 규칙 적용 → 등급 자동 분류 → 저장 → 결과 표시
```

**규칙:**

- 중복 리드 자동 필터링 (연락처 기준)
- 필수 컬럼 누락 시 에러 메시지 표시
- **등급 규칙에 해당하지 않는 리드는 기본 등급(설정 가능) 부여**
- 업로드 이력 저장 (날짜, 건수, 업로드자, 등급별 분류 결과)

**업로드 결과 화면 예시:**

```
✅ 업로드 완료: 총 150건

📊 자동 등급 분류 결과:
- A등급: 12건 (8%)
- B등급: 38건 (25%)
- C등급: 67건 (45%)
- D등급: 33건 (22%)

⚠️ 중복 제외: 5건
❌ 오류: 2건 (상세보기)
```

**예외 처리:** 잘못된 형식의 CSV 업로드 시 오류 행 표시 및 재업로드 안내

> 💡 **설정 연동:** CSV 컬럼 매핑 및 등급 규칙은 F03, F14에서 관리

#### F02. 리드 전체 목록 조회 및 필터링

**설명:** 업로드된 모든 리드를 테이블 형태로 조회

**화면 구성:**

- 테이블 컬럼: 신청일, 업체명, 대표자명, 연락처, 업종, 연매출, **등급 (색상 표시)** , 담당 팀장, 소속 팀, 상태
- 필터: 기간, 등급, 팀, 담당 팀장, 상태 (미배분/배분완료/컨택중/완료)

- **등급 일괄 변경: 복수 선택 후 등급 일괄 수정 가능**

등급을 추가/수정/삭제하고 각 등급의 기본 정보를 설정

| 설정 항목      | 설명                                       | 예시                               |
| -------------- | ------------------------------------------ | ---------------------------------- |
| 등급명         | 등급 이름                                  | A, B, C, D 또는 VIP, 일반, 보류 등 |
| 등급 설명      | 등급에 대한 설명 텍스트                    | "고매출 대규모 업체"               |
| 표시 색상      | 목록에서 표시될 배경/텍스트 색상           | 초록, 파랑, 노랑, 회색             |
| 우선순위       | 등급 정렬 순서 (숫자가 낮을수록 높은 등급) | 1, 2, 3, 4                         |
| 기본 등급 여부 | 규칙에 해당하지 않을 때 부여할 등급        | D등급을 기본값으로                 |

**3-2. 등급 규칙 설정**

각 등급에 해당하는 조건을 설정. 조건은 AND/OR로 조합 가능

**사용 가능한 조건 필드:**

| 필드명        | 데이터 타입 | 사용 가능한 연산자                     |
| ------------- | ----------- | -------------------------------------- |
| 연매출        | 숫자        | =, ≠, >, ≥, <, ≤, 범위 (A 이상 B 미만) |
| 종업원수      | 숫자        | =, ≠, >, ≥, <, ≤, 범위                 |
| 업종          | 텍스트      | =, ≠, 포함, 미포함, 목록에서 선택      |
| 지역          | 텍스트      | =, ≠, 포함, 미포함                     |
| 광고 캠페인   | 텍스트      | =, ≠, 포함, 미포함                     |
| 기타 CSV 필드 | 자동 매핑   | 필드 타입에 따라                       |

**규칙 설정 UI 예시:**

```
┌─────────────────────────────────────────────────────────────┐
│ 📋 A등급 규칙                                    [+ 조건 추가] │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  조건 1: [연매출 ▼] [이상 ▼] [10] [억원]                      │
│                                                  [AND ▼]    │
│  조건 2: [종업원수 ▼] [이상 ▼] [50] [명]                      │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│  📝 설명: 연매출 10억 이상 AND 종업원 50명 이상               │
│                                                             │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ 📋 B등급 규칙                                    [+ 조건 추가] │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  조건 1: [연매출 ▼] [범위 ▼] [5] ~ [10] [억원]                │
│                                                  [OR ▼]     │
│  조건 2: [종업원수 ▼] [범위 ▼] [20] ~ [50] [명]               │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│  📝 설명: 연매출 5-10억 OR 종업원 20-50명                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**규칙 적용 순서:**

1. 등급은 우선순위 순서대로 평가 (A → B → C → D)
2. 가장 먼저 조건을 만족하는 등급이 부여됨
3. 어떤 규칙에도 해당하지 않으면 기본 등급 부여

**3-3. 등급 규칙 테스트**

설정한 규칙이 올바르게 작동하는지 테스트

**테스트 방법:**

- 샘플 데이터 입력하여 어떤 등급이 부여되는지 확인
- 기존 리드 데이터에 규칙 재적용 시뮬레이션
- 예상 분류 결과 미리보기

```
┌─────────────────────────────────────────────────────────────┐
│ 🧪 규칙 테스트                                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  테스트 데이터 입력:                                         │
│  • 연매출: [12] 억원                                        │
│  • 종업원수: [35] 명                                        │
│  • 업종: [제조업 ▼]                                         │
│                                                             │
│  [테스트 실행]                                               │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│  ✅ 결과: A등급                                             │
│  📝 적용된 규칙: 연매출 10억 이상 AND 종업원 50명 이상        │
│     → 연매출 조건 충족 (12억 ≥ 10억)                        │
│     → 종업원수 조건 미충족 (35명 < 50명)                     │
│     → A등급 규칙 미충족, B등급으로 재평가...                  │
│     → B등급 조건 충족 (5억 ≤ 12억 ≤ 10억? NO)               │
│     → (수정 필요: 실제로는 12억은 B 범위 초과)               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**3-4. 기존 리드 등급 재분류**

규칙 변경 후 기존 리드에 새 규칙을 적용할지 선택

| 옵션                    | 설명                                                    |
| ----------------------- | ------------------------------------------------------- |
| 신규 리드에만 적용      | 이후 업로드되는 리드에만 새 규칙 적용                   |
| 전체 재분류             | 기존 모든 리드에 새 규칙 재적용 (수동 변경한 등급 포함) |
| 자동 분류된 것만 재분류 | 수동으로 변경한 등급은 유지, 자동 분류된 것만 재적용    |

**주의사항:**

- 전체 재분류 시 확인 팝업 표시
- 재분류 전 현재 등급 분포와 예상 변경 결과 미리보기 제공
- 재분류 이력 저장

---

**3-5. 기본 등급 규칙 예시**

| 등급 | 조건                              | 설명                  |
| ---- | --------------------------------- | --------------------- |
| A    | 연매출 ≥ 10억 AND 종업원 ≥ 50명   | 대규모 고매출 업체    |
| B    | 연매출 5억~10억 OR 종업원 20~50명 | 중규모 성장 업체      |
| C    | 연매출 1억~5억 OR 종업원 5~20명   | 소규모 업체           |
| D    | 위 조건에 해당하지 않음 (기본값)  | 정보 부족 / 소형 업체 |

#### F04. 리드 등급 수동 변경

**설명:** 자동 분류된 등급을 관리자가 수동으로 변경

**방법:**

- 리드 목록에서 각 행의 등급 드롭다운 선택하여 변경
- 복수 선택 후 일괄 등급 변경 가능

**규칙:**

- 수동 변경 시 변경 이력 저장 (변경자, 변경 일시, 이전 등급, 변경 등급)
- **수동 변경 여부 표시 (자동 분류 / 수동 변경 구분)**
- 등급 재분류 시 수동 변경한 등급 보호 옵션 제공

#### F05. 팀장(컨설턴트) 관리

**설명:** 리드를 배분받을 팀장 목록을 관리

**기능:**

- 팀장 등록: 이름, 이메일, 연락처, 로그인 ID/PW, 소속 팀 지정
- 팀장 수정: 정보 변경, 비밀번호 초기화, 소속 팀 변경
- 팀장 비활성화: 퇴사/휴직 시 배분 대상에서 제외 (삭제 아님)
- 팀장 목록: 이름, 소속 팀, 현재 배분 리드 수, 이번 달 실적 표시

#### F06. 팀 및 영업 관리자 관리

**설명:** 영업 조직의 팀 구조와 영업 관리자를 관리

**팀 관리 기능:**

- 팀 등록: 팀명, 팀 설명
- 팀 수정: 팀명 변경
- 팀 비활성화: 팀 해체 시 (소속 팀장 재배치 필요)
- 팀 목록: 팀명, 영업 관리자, 소속 팀장 수, 팀 전체 리드 수

**영업 관리자 관리 기능:**

- 영업 관리자 등록: 이름, 이메일, 연락처, 로그인 ID/PW, 담당 팀 지정 (복수 가능)
- 영업 관리자 수정: 정보 변경, 담당 팀 변경
- 영업 관리자 비활성화

**규칙:**

- 한 팀에는 최소 1명의 영업 관리자 필요
- 영업 관리자는 여러 팀을 담당할 수 있음
- 팀장은 반드시 하나의 팀에 소속되어야 함

#### F07. 드롭다운으로 팀장 배분 ⭐ 핵심 기능

**설명:** 시스템 관리자가 리드 목록에서 각 리드에 담당 팀장을 지정

**방법:**

- 각 리드 행에 '담당 팀장' 드롭다운 표시
- 드롭다운에 등록된 팀장 이름 목록 표시 (팀별로 그룹핑하여 표시)
- 팀장 선택 시 즉시 저장 (자동 저장)
- 복수 리드 선택 → 일괄 배분 기능

**규칙:**

- 팀장 배분 시 해당 팀장의 개인 페이지에 리드 즉시 노출
- 배분 이력 기록 (배분자, 배분 일시, 팀장)
- 재배분 가능 (담당 팀장 변경 시 이력 유지)
- 배분 시 해당 팀의 영업 관리자 대시보드에도 반영

#### F08. 팀장별 개인 페이지

**설명:** 팀장 로그인 시 본인에게 배분된 리드만 확인

**화면 구성:**

- 신규 리드: 오늘 배분받은 리드 (강조 표시)
- 미처리 리드: 아직 연락하지 않은 리드
- 진행 중 리드: 컨택 완료 / 미팅 예정
- 완료 리드: 계약 성사 / 실패
- 내 실적 요약: 이번 달 컨택률, 미팅률, 계약 성사 수
- 동행 요청 현황: 내가 보낸 요청 / 받은 요청

> 💡 **설정 연동:** 리드 상태 분류 기준은 시스템 설정(F14)의 상태값과 연동

#### F09. 리드 컨택 결과 입력

**설명:** 팀장이 리드 컨택 후 결과를 입력

**입력 항목 (기본값, 설정에서 변경 가능):**

| 항목           | 기본 옵션                                         | 설정 가능 여부         |
| -------------- | ------------------------------------------------- | ---------------------- |
| 컨택 상태      | 미연락 / 통화 성공 / 부재 / 번호 오류 / 연락 거부 | ✅ 추가/수정/삭제 가능 |
| 미팅 상태      | 해당없음 / 미팅 예정 / 미팅 완료 / 미팅 취소      | ✅ 추가/수정/삭제 가능 |
| 계약 상태      | 해당없음 / 상담 중 / 계약 성사 / 계약 실패        | ✅ 추가/수정/삭제 가능 |

| 다음 연락 일정 | 날짜/시간 선택                                    | ❌ 고정                |
| 미팅 일정      | 날짜/시간/장소 입력                               | ❌ 고정                |

**규칙:**

- 계약 성사 시 계약 금액 필수 입력

- 미팅 예정 상태 시 미팅 일정 필수 입력

> 💡 **설정 연동:** 컨택/미팅/계약 상태 옵션은 시스템 설정(F14)에서 관리

#### F10. 영업 관리자 팀 대시보드

**설명:** 영업 관리자가 담당 팀의 현황을 모니터링하는 전용 대시보드

**화면 구성:**

| 영역           | 표시 내용                                                       |
| -------------- | --------------------------------------------------------------- |
| 팀 요약        | 팀 전체 리드 수, 금주 신규 배분, 미처리 리드 수,**등급별 분포** |
| 팀원별 현황    | 각 팀장의 보유 리드 수, 컨택률, 미팅률, 계약률, 오늘 활동 여부  |
| 팀 파이프라인  | 팀 전체 리드 → 컨택 → 미팅 → 계약 단계별 수치                   |
| 팀 실적 추이   | 주간/월간 팀 전체 계약 성사 그래프                              |
| 금주 미팅 일정 | 팀원들의 예정된 미팅 캘린더 뷰                                  |
| 동행 요청 현황 | 팀 내/팀 간 동행 요청 목록 및 상태                              |

**기능:**

- 팀원 클릭 시 해당 팀장의 상세 리드 목록 조회 (읽기 전용)
- 기간 필터: 오늘 / 이번 주 / 이번 달
- Excel 내보내기: 팀 현황 리포트

**권한:**

- 담당 팀의 데이터만 조회 가능
- 다른 팀 데이터는 접근 불가

#### F11. 미팅 동행 요청

**설명:** 팀장이 다른 팀장에게 미팅 동행을 요청하는 기능

**요청 프로세스:**

```
요청자 → [동행 요청] → 수신자 → [수락/거절] → 결과 확정
```

**요청 생성 (요청자):**

- 리드 상세 화면에서 '동행 요청' 버튼 클릭
- 입력 항목:
  - 동행 요청할 팀장 선택 (전체 팀장 목록에서 검색)
  - 미팅 일시 (기 입력된 미팅 일정 자동 표시)
  - 미팅 장소
  - 요청 사유 / 메모 (선택)

**요청 수신 (수신자):**

- 개인 페이지에 '받은 동행 요청' 알림 표시
- 요청 상세 확인 가능: 요청자, 리드 정보, 미팅 일시/장소, 요청 사유
- 응답: 수락 / 거절 (거절 시 사유 입력)

**상태 관리:**

| 상태   | 설명                               |
| ------ | ---------------------------------- |
| 대기중 | 요청 후 수신자 응답 전             |
| 수락됨 | 수신자가 동행 수락                 |
| 거절됨 | 수신자가 동행 거절                 |
| 완료   | 미팅 완료 후 (자동 또는 수동 변경) |
| 취소됨 | 요청자가 요청 취소                 |

**알림:**

- 요청 시: 수신자에게 알림
- 수락/거절 시: 요청자에게 알림
- 미팅 1일 전: 양측에게 리마인더

**이력 관리:**

- 모든 동행 요청 이력 저장
- 리드별 동행 이력 조회 가능
- 팀장별 동행 참여 통계 (요청 횟수, 수락률 등)

**권한:**

- 모든 팀장이 모든 팀장에게 요청 가능 (같은 팀, 다른 팀 무관)
- 영업 관리자는 팀 내 동행 요청 현황 조회 가능
- 시스템 관리자는 전체 동행 요청 현황 조회 가능

#### F12. 시스템 관리자 실시간 대시보드

**설명:** 사무직원/경영진용 전체 현황 모니터링

**주요 위젯:**

- 오늘의 요약: 신규 리드 수, 배분 완료 수, 미배분 수
- **등급별 분포: 전체 리드의 등급 분포 차트**
- 팀별 현황: 각 팀의 리드 수, 컨택률, 미팅률, 계약률
- 팀장별 현황: 각 팀장의 보유 리드 수, 컨택률, 미팅률, 계약률
- 파이프라인 퍼널: 전체 리드 → 컨택 → 미팅 → 계약 단계별 수치
- 기간별 추이: 주간/월간 리드 유입 및 계약 성사 그래프
- 광고 캠페인별 리드 현황: 어떤 광고에서 몇 건 유입되었는지
- 동행 요청 현황: 전체 동행 요청 통계

**기능:**

- 필터: 기간, 팀, 팀장, 등급, 광고 캠페인
- 드릴다운: 숫자 클릭 시 해당 리드 목록으로 이동
- 자동 새로고침: 1분 간격 (설정 가능)

#### F14. 시스템 설정 관리

**설명:** 운영 중 변경될 수 있는 값들을 시스템 관리자가 직접 설정할 수 있는 기능

**설정 가능 항목:**

| 카테고리        | 설정 항목              | 설명                                   | 기본값                    |
| --------------- | ---------------------- | -------------------------------------- | ------------------------- |
| **등급 관리**   | 등급 정의              | 등급 추가/수정/삭제, 색상, 우선순위    | A, B, C, D                |
|                 | **등급 규칙**          | 자동 분류 조건 설정 (F03 참조)         | (기본 규칙)               |
|                 | 기본 등급              | 규칙 미해당 시 부여할 등급             | D                         |
| **상태값 관리** | 컨택 상태              | 컨택 상태 옵션 추가/수정/삭제/순서변경 | 미연락, 통화성공, 부재 등 |
|                 | 미팅 상태              | 미팅 상태 옵션 추가/수정/삭제/순서변경 | 해당없음, 미팅예정 등     |
|                 | 계약 상태              | 계약 상태 옵션 추가/수정/삭제/순서변경 | 해당없음, 상담중 등       |
|                 | 최종 상태 지정         | 어떤 상태가 "완료"로 간주될지 지정     | 계약성사, 계약실패        |
| **동행 요청**   | 요청 상태              | 동행 요청 상태 옵션 관리               | 대기중, 수락됨, 거절됨 등 |
|                 | 리마인더 설정          | 미팅 전 알림 시점                      | 1일 전                    |
| **CSV 매핑**    | 컬럼 매핑              | CSV 컬럼명 ↔ 시스템 필드 매핑          | Meta 기본 형식            |
|                 | 필수 컬럼              | 업로드 시 필수로 있어야 하는 컬럼      | 이름, 연락처              |
| **업종 관리**   | 업종 목록              | 업종 선택 옵션 관리                    | (자유 입력 또는 목록)     |
| **기타**        | 대시보드 새로고침 주기 | 자동 새로고침 간격                     | 1분                       |
|                 | 세션 타임아웃          | 자동 로그아웃 시간                     | 8시간                     |

## 5. 비기능 요구사항 (Non-functional Requirements)

### 5.1 성능

- 페이지 로딩 속도: 3초 이내 (초기 로딩), 1초 이내 (페이지 전환)
- 동시 접속자: 최대 50명 처리 가능
- CSV 업로드: 1,000건 기준 10초 이내 처리 **(등급 자동 분류 포함)**
- 대시보드 데이터 갱신: 1분 이내
- **등급 규칙 적용: 리드 1건당 10ms 이내**

### 5.2 보안

- HTTPS 필수 적용
- 사용자 인증: JWT 기반 토큰 인증
- 역할 기반 접근 제어 (RBAC): 시스템 관리자 / 영업 관리자 / 팀장 권한 분리
- 비밀번호: bcrypt 해싱, 최소 8자 이상
- 개인정보: 연락처 마스킹 표시 옵션
- 접근 로그: 모든 데이터 접근/변경 기록
- 세션 관리: 8시간 후 자동 로그아웃 (설정 변경 가능)

### 5.3 접근성

- 반응형 웹 디자인 (데스크톱, 태블릿 지원)
- 최소 지원 해상도: 1280x720
- 한국어 인터페이스

### 5.4 호환성

- 브라우저: Chrome 90+, Safari 14+, Edge 90+
- CSV 파일: UTF-8, EUC-KR 인코딩 지원

### 5.5 가용성

- 서비스 가용성: 99% (월간)
- 정기 점검: 매주 일요일 새벽 2-4시
- 데이터 백업: 일 1회 자동 백업, 30일 보관
